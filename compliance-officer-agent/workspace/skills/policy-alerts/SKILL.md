# Policy Alerts Skill

Generate and route compliance alerts to appropriate stakeholders.

## When to Use

- When compliance issues are detected
- When deadlines are approaching
- When regulatory changes occur
- During incident response

## Alert Types

### 1. Critical Alerts

**Trigger**: Immediate action required
- Data breach suspected
- Compliance deadline missed
- Critical audit finding
- Regulatory enforcement action

**Channels**: Phone + Slack + Email (simultaneous)
**Response Time**: Immediate

### 2. High Priority Alerts

**Trigger**: Action within 24 hours
- Deadline within 7 days
- High-severity finding
- Policy violation detected
- New critical regulation

**Channels**: Slack + Email
**Response Time**: Within 24 hours

### 3. Medium Priority Alerts

**Trigger**: Action within 7 days
- Deadline within 30 days
- Medium-severity finding
- Upcoming audit

**Channels**: Email
**Response Time**: Within 7 days

### 4. Low Priority Alerts

**Trigger**: Informational
- Weekly summary
- Completed tasks
- Scheduled reminders

**Channels**: Weekly report
**Response Time**: Within 1 week

## Alert Format

```
🚨 [ALERT_TYPE] - [Title]

**Priority**: [Critical/High/Medium/Low]
**Regulation**: [Applicable regulation]
**Deadline**: [Date if applicable]
**Owner**: [Responsible party]

## Description
[Detailed description of the issue/requirement]

## Recommended Action
1. [Step 1]
2. [Step 2]
3. [Step 3]

## Related Information
- Source: [Link]
- Reference: [Document/ID]

---
Generated by Compliance Officer Agent
```

## Routing Rules

| Alert Type | Primary Channel | Backup Channel | Escalation |
|------------|-----------------|----------------|------------|
| Critical | Slack #critical | Phone | CISO + CEO |
| High | Slack #compliance | Email | Compliance Lead |
| Medium | Email | Slack | Team Lead |
| Low | Weekly Report | - | - |

## Configuration

Set alert preferences via:

```json
{
  "compliance": {
    "alerts": {
      "critical": {
        "slack": "#critical-alerts",
        "email": "ciso@company.com,ceo@company.com"
      },
      "high": {
        "slack": "#compliance",
        "email": "compliance@company.com"
      },
      "medium": {
        "email": "compliance@company.com"
      }
    }
  }
}
```

```
**Owner**: [NAME]
**Required Action**: [DESCRIPTION]

```

### Audit Finding Template

```
🔍 Audit Finding Logged

**Severity**: [Critical/High/Medium/Low]
**Finding**: [Description]
**Root Cause**: [If known]
**Remediation Required**: [Yes/No]
**Due Date**: [If applicable]
**Owner**: [NAME]

Next Steps:
1. [Action]
2. [Action]
```

## Best Practices

- Always include actionable next steps
- Provide links to full documentation
- Assign clear ownership
- Set realistic deadlines
- Follow up on unresolved alerts
- Document all alert resolutions
